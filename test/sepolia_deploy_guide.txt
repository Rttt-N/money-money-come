Sepolia 测试网部署指南 — MoneyMoneyCome（混合模式）
====================================================
部署方案：真实 Chainlink VRF + 真实 Chainlink Automation + MockAave + MockUSDC
所有资产（ETH、LINK）为测试代币；USDC 由 MockUSDC 合约自由 mint。

第五部分：Chainlink Automation 注册（浏览器 + 终端）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Automation 的作用：每当一轮游戏到期，Chainlink 节点自动调用 performUpkeep 触发开奖，
无需手动运行脚本或等待 demo.ts。

步骤 8 — 注册 Upkeep（浏览器）
  网址：https://automation.chain.link/sepolia
  操作：
    1. 点击 Register New Upkeep
    2. Trigger 选择 Custom Logic
    3. Target contract address: 粘贴 MoneyMoneyCome 地址
    4. Upkeep name: MoneyMoneyCome（自定义）
    5. Gas limit: 填 500000（performUpkeep 自身的 gas，不含 VRF 回调）
    6. Starting balance: 填 2（LINK）
    7. 点击 Register upkeep → 确认两笔交易（approve LINK + register）
    8. 完成后记录页面 URL 末尾的 Upkeep ID（纯数字）

步骤 9 — 绑定 Forwarder 地址到合约（终端）
  注册成功后，Chainlink 会为这个 Upkeep 分配一个专属 Forwarder 地址。
  需要把这个地址绑定到合约，使只有 Chainlink 和 owner 可以触发开奖。

  方式一（推荐）：提供 Upkeep ID，脚本自动从 Registry 读取 Forwarder 并绑定
    UPKEEP_ID=你的UpkeepID npx hardhat run scripts/set-automation-forwarder.ts --network sepolia

  方式二（手动）：在 automation.chain.link 找到 Upkeep → 复制 Forwarder address 后运行
    FORWARDER=0x你的Forwarder地址 npx hardhat run scripts/set-automation-forwarder.ts --network sepolia

  方式三（清除绑定，恢复任何人均可调用的演示模式）：
    FORWARDER=0x0000000000000000000000000000000000000000 npx hardhat run scripts/set-automation-forwarder.ts --network sepolia

  预期输出：
    MoneyMoneyCome       : 0x...
    Current forwarder    : (not set — open mode)
    Setting forwarder to : 0x...（Chainlink Forwarder）
    ✓ Done.
      Mode: RESTRICTED — only Chainlink Automation forwarder or owner

  ⚠ 绑定后，只有 Chainlink Automation 节点和合约 owner 可以触发开奖。
  ⚠ 每次重新部署合约后，需重新注册 Upkeep 并重新绑定 Forwarder。

步骤 10 — Mint MockUSDC 到自己的钱包
  npx hardhat run scripts/mint-usdc.ts --network sepolia

  默认 mint 100,000 USDC 到部署者账户。
  自定义金额和目标：
    MINT_TO=0x你的地址 MINT_AMOUNT=5000 npx hardhat run scripts/mint-usdc.ts --network sepolia

  PowerShell 写法：
    $env:MINT_TO="0x你的地址"; $env:MINT_AMOUNT="5000"
    npx hardhat run scripts/mint-usdc.ts --network sepolia

步骤 11 — 在 MetaMask 中显示 MockUSDC 余额
  1. 打开 MetaMask → 确保在 Sepolia 网络
  2. 点击 "Assets" 标签页底部的 "Import tokens"
  3. Token contract address: 粘贴步骤 6 输出的 MockUSDC 地址
  4. Token symbol: 自动填充为 "USDC"
  5. Token decimal: 自动填充为 "6"
  6. 点击 "Add Custom Token" → "Import Tokens"

  现在 MetaMask 中可以看到你的 MockUSDC 余额了。

步骤 12 — 运行演示脚本（全流程：存入 → 等待开奖 → 显示结果）
  npx hardhat run scripts/demo.ts --network sepolia

  可选参数：
    DEPOSIT_USDC=200 TIER=2 YIELD_USDC=100 npx hardhat run scripts/demo.ts --network sepolia

  演示流程说明：
    1. 读取已部署合约地址
    2. 检查 USDC 余额，不足时自动 Mint
    3. Approve + enterGame（链上交易，约 15-30 秒确认）
    4. 模拟 Aave 收益（+50 USDC，可通过 YIELD_USDC 自定义）
    5. 等待轮次到期（默认 5 分钟）
    6. 调用 performUpkeep，发送 VRF 请求
       （接入 Automation 后，此步骤由 Chainlink 自动完成）
    7. 每 10 秒轮询，等待 Chainlink VRF 回调（约 1-3 分钟）
    8. 显示获奖者和奖金

  注意：自动生息已启用（2%/分钟），5 分钟内自动累积约 10% 利息到奖池。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

如果轮次卡在 DRAWING 或 LOCKED 状态超过 10 分钟（VRF 回调失败）：

  # 查看当前状态并强制结算（无赢家），自动开启新一轮
  npx hardhat run scripts/emergency-reset.ts --network sepolia

  效果：
  - 当轮标记为 SETTLED（无赢家，奖池归零）
  - 所有参与者 principal 自动 rollover 到新一轮（本金安全）
  - 新一轮立即以 OPEN 状态开始

  解决 VRF gas 不足问题（执行 emergency-reset 后操作）：
  GAS_LIMIT=1500000 npx hardhat run scripts/set-callback-gas.ts --network sepolia

  ⚠ emergencyReset() 只有合约 owner（部署钱包）可以调用

常见问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: 部署失败，提示 insufficient funds
A: 钱包 Sepolia ETH 不足，去 https://faucets.chain.link/sepolia 领取。

Q: performUpkeep 报错 0x5247fdce（InvalidConsumer）
A: MoneyMoneyCome 未添加为 VRF Subscription 的 Consumer。
   去 vrf.chain.link/sepolia → 找到订阅 → Add Consumer → 粘贴合约地址。

Q: performUpkeep 报错 MMC: not authorized
A: automationForwarder 已绑定，当前调用者不是 Forwarder 或 owner。
   检查 automation.chain.link 中 Forwarder 地址是否正确，
   或用 owner 钱包重新运行 set-automation-forwarder.ts。

Q: VRF 报错 gas limit too low / callback failed
A: 运行：GAS_LIMIT=1500000 npx hardhat run scripts/set-callback-gas.ts --network sepolia
   无需重新部署，立即生效。若已卡在 DRAWING 状态，先 emergency-reset 再重试。

Q: demo 等待 VRF 超过 10 分钟
A: 1. 检查 vrf.chain.link/sepolia：Consumer 是否已添加？LINK 是否充足（≥1 LINK）？
   2. 运行 emergency-reset.ts 解除卡死，提高 callbackGasLimit 后重试。

Q: 轮次卡在 DRAWING / LOCKED 无法开新游戏
A: 运行 npx hardhat run scripts/emergency-reset.ts --network sepolia
   强制结算当前轮次，新一轮自动开启。

Q: 前端显示"No deployment found"
A: 确认 frontend/lib/addresses.json 中有 "11155111" 的条目。
   重新运行部署脚本即可自动更新。

Q: 如何给其他测试账户 mint USDC？
A: MINT_TO=0x目标地址 npx hardhat run scripts/mint-usdc.ts --network sepolia

Q: 开奖后奖金为 0
A: 确保在开奖前有利息累积：
   1. 自动利息：部署时已设置 2%/分钟利率，5 分钟自动累积约 10%
   2. 手动注入：YIELD_AMOUNT=50 npx hardhat run scripts/simulate-yield.ts --network sepolia

Q: MetaMask 中看不到 USDC 余额
A: 需要手动导入代币。MetaMask → Import tokens → 粘贴 MockUSDC 合约地址。

Q: 需要重新部署（修改合约后）
A: 直接重新运行 deploy.ts，addresses.json 自动更新。
   重新部署后需要：
   1. 重新添加 VRF Consumer（vrf.chain.link/sepolia）
   2. 重新注册 Automation Upkeep（automation.chain.link/sepolia）或更新 Target 地址
   3. 重新运行 set-automation-forwarder.ts 绑定新 Forwarder

Q: 本项目和真实 Aave 的关系是什么？
A: 本项目使用 MockAave（带自动生息功能）模拟 Aave V3 的存取接口。
   合约架构（YieldVault → AavePool → aToken）与真实 Aave 完全一致，
   只需替换地址即可切换到真实 Aave。Chainlink VRF 和 Automation 是真实的。
